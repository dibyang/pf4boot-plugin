# 安全报告

[中文](SECURITY.md) | [English](docs/en/SECURITY.md)

感谢你帮助维护 pf4boot-plugin 的安全性。请不要在公开 issue、讨论区或 PR 中披露可利用细节、密钥、内部路径或可直接复现攻击的步骤。

## 如何报告

优先使用 GitHub Private Vulnerability Reporting：

<https://github.com/dibyang/pf4boot-plugin/security/advisories/new>

如果该入口不可用，请创建一个不包含细节的公开 issue，占位说明“需要私下报告安全问题”，维护者会协助建立私下沟通渠道。

## 请包含的信息

私下报告时请尽量提供：

- 受影响版本或 commit。
- 受影响组件，例如 `pf4boot` 打包任务、依赖解析、发布任务或本地运行 classpath。
- 漏洞类型和影响范围。
- 最小复现步骤或概念验证。
- 是否已经在真实环境中触发。
- 可能的缓解方式或修复建议。

## 安全敏感示例

以下问题请按安全报告处理，不要公开细节：

- 任意文件读取、写入或路径穿越。
- 敏感信息泄露，例如凭据、Token、私钥或内部构建路径。
- 可利用的反序列化、脚本执行或命令注入。
- 权限绕过、发布凭据滥用或供应链投毒。
- 可远程触发的拒绝服务。

## 处理预期

维护者会尽快确认报告，并根据影响范围安排修复、回归测试和发布。公开披露时间应在修复版本可用后再协商确定。
